About

Cloud users and service providers are increasingly concerned about the management of their data and the behavior of the applications they use/own once stored/deployed in the cloud. They therefore ask for enhanced assurance solutions, which partially mitigate the new risks and threats they are facing. Among existing solutions, certification has been widely adopted as a preferable approach to increase trust in the cloud. In this paper, after briefly discussing our test-based certification scheme for the cloud, we show a real certification process aimed to certify OpenStack, an open source IaaS solution for managing infrastructure resources. In particular, we first describe the testing activities executed to certify OpenStack for security and performance properties. We then illustrate the obtained results and the outcomes of the certification process.

Test Agent - Installation

Developer/Test:

  • Get a Centos 7 up and running
  • Create user ta: 
    adduser ta
  • User ta needs to have sudo privileges: 
    yum install -y sudo
echo "stack ALL=(ALL) NOPASSWD: ALL" >> /etc/sudoers
  • Checkout the git at: 
    git checkout https://github.com/fgaudenzi/mooncloud.git
  • Launch the installation script: 
    ./installer.sh
  • Check everything is working 
    cd ~
python probe.py test.xml

Deploy directly on Openstack:

  • Download offical snapshot from below
  • Load the snapshot in Glance
  • Deploy a new VM in Openstack using the snapshot
Download Test Agent OpenStack Snapshot

Usage

SESAR Lab have already devloped several probes, which covers different requirements over the whole cloud stack layers:

  • Infrastructure: OpenStack Connectivity over TLS/SSL, file access management, OpenStack Security Storage (Swift), OpenStack network isolation enforcement (active test for security group), OpenStack Tempest integration, Automated PenTesting, Openstack Vulnerability Assesment
  • Platform: Connectivity over TLS/SSL, file access management, MySql-adhoc rows integrity checker, vulnerability assessment, docker vulnerability checker
  • Software: OWASP top 10 compliancy, sql-injection, delay after n loging attempts test, ad-hoc tests, Nessus integration

Test Agent comes with Channel Encryption probe and OpenStack network isolation probe already configured to be executed.
Run Channel Encryption probe:

  • Copy the sample in sample/encchannel.xml in ta home folder 
    cp sample/encchannel.xml ~/
  • Open encchannel.xml with a texteditor and replace the fields with your configuration as follow:
    $host
$ip
$port
$encryption
